Stellar-Wiki完全使用指南

本文记录了在 Stellar 主题中搭建 Wiki（知识库）的完整流程、文件结构和常见坑，方便下次独立操作。

一、Wiki 是什么

Stellar 的 Wiki 是一个结构化知识库系统，和”博客文章”的区别：

博客文章（_posts/） Wiki（wiki/）
组织方式 按时间倒序 按树形目录（tree）
侧边栏 相关文章、最新文章 目录树 + 相关 + 最新
适合放什么 碎碎念、踩坑记录 系统性笔记、专题知识

Wiki 首页（/wiki/）会展示所有上架项目的卡片，点击进入具体项目后可浏览该项目的所有页面。

字段说明：

字段 必填 说明
title ✅ 显示在 Wiki 首页卡片和浏览器标题栏
name ✅ 短名称，用于导航
tags 可选 标签数组，用于 Wiki 标签页（/wiki/tags/设计模式/）
base_dir ✅ 关键！ 项目页面的基础路径，必须和 source/wiki/ 下的路径一致。末尾要有 /
tree ✅ 目录树结构。key 是分组标题（'' 表示不分组），value 是页面 key 数组
sort 可选 数字，越大排越前，默认 0
homepage 可选 自定义首页路径，不填则自动取 tree 的第一个页面

七、常见问题排查

现象 可能原因 解决
/wiki/ 首页空白 source/_data/wiki.yml 不存在或为空 创建文件并写入项目 ID 列表
首页有卡片但点进去没页面 页面的 wiki: 字段缺失或写错 检查 front-matter 中 wiki: 是否与 _data/wiki/<id>.yml 的 id 一致
页面显示 0 字节 layout: wiki 写错了 改为 layout: page
左侧没有目录树 leftbar 没配 tree 检查 site_tree.wiki.leftbar
页面在 ... 分组里 tree 配置漏了这个页面 在项目配置的 tree 中补充该页面的 key
目录树顺序不对 tree 中的 key 顺序即显示顺序 调整项目 yml 中 key 的排列顺序
构建时有 WARN No layout 同上，layout 字段写错 改为 layout: page

九、与 Hexo 标准博客的对比

这张表帮你记住什么时候用博客文章、什么时候用 Wiki：

维度 博客文章 Wiki 页面
创建命令 hexo new post "xxx" 手动创建 .md 文件
存放位置 source/_posts/ source/wiki/<项目ID>/
默认 layout post page（不是 wiki！）
关联字段 无 wiki: <项目ID>
URL /2026/07/23/title/ /wiki/<项目ID>/page/
归档 出现在 /archives/ 出现在 wiki 项目目录树
标签 全局标签系统 全局标签 + wiki 专属标签页
适合内容 时间敏感、碎片化 体系化、需长期维护
