MewCode 第一阶段 — 核心概念与 Claude Code 实战笔记

发表于 2026-06-25

一、这个阶段在做什么

第一阶段的目标是建立对 Coding Agent 的整体认知。我们从最核心的概念出发——Agent 到底是什么，然后拆解一个 Coding Agent（MewCode）的架构分层，最后通过 Claude Code 的实战技巧来理解一个成熟的 Coding Agent 是怎么用的、能做什么、以及怎么用好它。

二、Agent 是什么

2.1 一句话定义

Agent = LLM + 循环 + 反馈 + 工具

拆开来看：

要素 含义 类比
LLM 大语言模型，是 Agent 的「大脑」 人的思考能力
循环（Loop） 不是一问一答就结束，而是持续地「思考→行动→观察→再思考」 人做事时反复试、反复调整
反馈（Feedback） 每执行一个动作，拿到结果，用来修正下一步 眼睛看到结果，告诉大脑要不要继续
工具（Tools） 读写文件、执行命令、搜索代码……这些是 Agent 的「手」 人的手和工具

2.2 Agent 的工作循环

2.3 本项目：MewCode

MewCode 是一个 Coding Agent——在终端里运行的 AI 编程助手，设计上参考了 Claude Code。它可以直接在你的项目里读写文件、执行命令、搜索代码，像一个坐在你旁边的 AI 程序员。

三、MewCode 五层架构

这是 MewCode 的顶层设计，每一层有明确的职责：

3.1 交互层（Interaction Layer）

职责：处理用户与 Agent 之间的对话。

  • 接收用户的输入（自然语言、命令）
  • 展示 Agent 的回复和执行过程
  • 管理对话历史和上下文窗口
  • 这是用户唯一能直接感知到的一层

3.2 引擎层（Engine Layer）

职责：Agent 的「大脑」，核心调度中心。

  • 调用 LLM 进行推理
  • 管理 Agent 的主循环（思考→行动→观察→再思考）
  • 决定什么时候该调用什么工具
  • 解析 LLM 的输出，转换为具体操作

3.3 工具层（Tool Layer）

职责：Agent 的「手脚」，执行具体操作。

工具类型 能力 典型场景
文件读写 读、写、编辑文件 修改代码、生成文档
命令执行 在终端执行 shell 命令 运行测试、安装依赖
代码搜索 正则搜索文件内容 定位函数、查找引用
MCP 扩展 通过 MCP 协议接入外部能力 搜网页、操作数据库

3.4 记忆层（Memory Layer）

职责：让 Agent 记住该记住的东西，跨对话保持上下文。

  • 项目记忆：CLAUDE.md / CLAUDE.local.md
  • 用户记忆：~/.claude/CLAUDE.md（跨项目生效）
  • 会话记忆：当前对话的上下文
  • 长期记忆：/memory 命令保存的跨项目记忆

3.5 安全层（Security Layer）

职责：保证 Agent 的操作是安全的、经过授权的。

  • 权限控制：哪些操作需要用户确认
  • 沙箱隔离：防止危险命令影响系统
  • 审计日志：记录 Agent 做了什么

四、Claude Code 实战技巧

以下是 Claude Code 的 9 大核心使用技巧，覆盖从入门到进阶的全部内容。

4.1 技巧一：用好 @ 文件引用

问题：Claude 不知道你的项目结构，会自己猜测文件位置，浪费上下文去「搜索」。

解法：用 @文件名 直接告诉它读哪个文件。

核心原则：精准投喂 > 让它自己找。上下文是有限资源，省着用。

4.2 技巧二：四种模式，按场景切换

Claude Code 有四种运行模式，控制自主程度从低到高：

模式 编辑文件 执行命令 适用场景
Plan ❌ 不动 ❌ 不动 复杂任务，先看方案再动手
Normal 每次询问 每次询问 默认模式，稳妥可靠
Accept Edits ✅ 自动 需要确认 信任它的代码能力，但命令要审
Auto ✅ 自动 ✅ 自动 完全放手，适合重复性任务

💡 建议：日常用 Normal，熟悉后编辑密集的任务用 Accept Edits，复杂的先开 Plan 看方案。

4.3 技巧三：后悔药——改错了怎么撤回

这是终端里的撤销机制，分两层：

第一层：快速撤回（适合小改动）

操作 效果
连续按两次 Esc，然后用 ↑ ↓ 选择 恢复到之前某个文件状态
输入 /rewind 命令 回到对话的某个历史节点

⚠️ 局限：只能恢复被修改的文件。如果 Agent 执行了某个命令（比如 rm -rf 或生成了大量新文件），撤回机制无能为力。

第二层：Git（适合大改动）

🎯 黄金法则：小改动用 Esc+Esc 或 /rewind，大改动用 Git。两者配合，万无一失。

4.4 技巧四：后台运行长时间任务

场景：测试、构建、安装依赖这些操作可能跑很久，不要让它占着对话。

用法：直接告诉 Claude「在后台运行」。

适用场景：

  • 项目构建（npm run build）
  • 跑测试套件
  • 安装大型依赖
  • 长时间的数据处理

💡 原理：后台任务跑完会自动通知你，期间你可以继续对话，不用干等。

4.5 技巧五：CLAUDE.md 记忆系统

问题：每次新对话，Claude 就像失忆了一样，需要重新了解你的项目。

解法：CLAUDE.md 是 Claude Code 的「项目记忆文件」。每次在这个项目启动 Claude，它会先读 CLAUDE.md，自动获得项目背景。

创建方式

或者手动在项目根目录创建 CLAUDE.md。

三级记忆体系

层级 文件位置 提交 Git 作用范围
项目级 项目根/CLAUDE.md 或 .claude/CLAUDE.md ✅ 提交 本项目所有人共享
个人项目级 项目根/CLAUDE.local.md ❌ .gitignore 只你一人，不干扰队友
用户级 ~/.claude/CLAUDE.md — 你所有的项目

怎么写 CLAUDE.md

🎯 核心原则：越精准越好。 不要写成 README，要写 Claude 真正需要知道的上下文。

4.6 技巧六：MCP——给 Claude 装外挂

Claude Code 内置能力：读写文件 + 执行命令 + 搜索代码。这三样对 Coding Agent 来说已经够用，但有时你需要更多。

MCP（Model Context Protocol） 就是 Claude 的「外设接口」——通过安装 MCP Server，给 Claude 接入新的能力。

常用 MCP Server 举例

MCP Server 能力 场景
Brave Search 搜索网页 查最新文档、找解决方案
Postgres 操作 PostgreSQL 直接查询/修改数据库
Playwright 控制浏览器 自动化测试、截图对比
Filesystem 访问指定目录外的文件 操作项目外的配置文件

4.7 技巧七：Skills 和 Hooks——自动化工作流

这是让 Claude Code 从「工具」变成「平台」的两个机制。

Skills（技能包）

是什么：可安装的功能模块，让 Claude 在特定领域表现更好。

示例：

拆解这条命令：

  • frontend-design：Skill 的名字，装完后 Claude 做前端设计时水平明显提升
  • claude-plugins-official：官方插件市场，Claude Code 内置，不需要额外 add

💡 类比你懂的：Skills 就像 VS Code 的插件，需要什么装什么。

Hooks（钩子）

是什么：在特定操作「之前」或「之后」自动触发你的自定义脚本。

典型场景：

Hook 时机 做什么
修改代码后 自动跑测试、lint
执行命令前 记录日志、备份文件
对话开始前 加载项目上下文

如何配置

配置文件： 项目根/.claude/settings.json（项目级）或 ~/.claude/settings.json（全局）。

⚠️ 重要细节：Hook 脚本不是通过环境变量拿文件路径的，而是通过 stdin 标准输入 接收一段 JSON，里面包含了这次操作的全部信息。你可以通过 $CLAUDE_PROJECT_DIR 拿到项目根目录，但具体改了哪个文件，需要从 stdin 的 JSON 里解析。

4.8 技巧八：子代理（Sub-Agent）

是什么：Claude 的「分身」——创建独立的 Agent 实例，有自己独立的上下文空间。

为什么要用子代理

原因 说明
审查更客观 相当于找另一个人看你的代码，不受主对话偏见影响
保护上下文 脏活累活交给分身干，主对话保持清爽
并行处理 多个子代理可以同时做不同的事

典型用法

场景 怎么做
代码审查 让子代理用独立视角审查代码质量
写测试 让子代理补齐单元测试，不受主对话干扰
查文档 让子代理去读某个库的文档，总结要点后汇报
批量修改 让子代理独立处理一组文件的修改

相关命令

🎯 一句话总结：子代理是 Claude 的「分身」，有独立的上下文空间，适合做代码审查、写测试等需要「独立视角」的任务。

4.9 技巧九：高频命令速查

命令 作用
/agents 列出所有子代理
/agents -d 列出子代理的详细信息
/agents -c 清理所有子代理的上下文
/context 查看当前用了多少上下文
/compact 压缩对话，释放上下文空间
/clear 清空对话
/init 初始化 CLAUDE.md
/mcp 进入 MCP 管理
/memory 管理跨项目记忆
/rewind 回退到对话历史节点
/tasks 查看后台任务
