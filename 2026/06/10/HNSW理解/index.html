HNSW理解

HNSW 算法理解

Hierarchical Navigable Small World — 分层可导航小世界图

核心思想：用多层图让搜索从”全局粗筛”到”局部精搜”，每一步都在图上贪心走，而不是暴力扫表。

基本原理

  1. HNSW 本质是多个层次的图堆叠在一起
  2. 每个节点有一个层级（random_level 决定），出现在该层及以下所有层
  3. 层级越高 → 该层的节点越稀疏（指数级减少）
  4. 每层内部是一个可导航小世界图（NSW）：每个节点有若干邻居，邻居之间形成短程 + 长程连接
  5. 搜索时从顶向下：高层做路由（快速定位区域），低层做精搜（找到具体近邻）
  6. 时间复杂度 O(log N)

数学原理

  • r ~ Uniform(0, 1] → -ln(r) 服从参数 λ=1 的指数分布
  • 乘以 mL_ = 1 / ln(M) → 缩放后取整，得到几何分布的层数
  • 层数 ≥ L 的概率：$P(level \ge L) = 1 / M^L$

实际效果（M=16）

层级 出现在该层的概率 约为
0 100% 所有节点
1 ~1/16 ≈ 6.25% 每 16 个中有 1 个
2 ~1/256 ≈ 0.39% 每 256 个中有 1 个
3 ~1/4096 ≈ 0.024% 每 4096 个中有 1 个

🔑 这就是”上层稀疏，下层密集”的来源。

从入口 ep 出发，沿着当前层的图边”贪心 BFS”，找到离 q 最近的 ef 个节点。

两个堆的分工

提前终止条件的直觉

每轮从 candidates 中取出最近未探索节点。如果它已经比 results 中最差的结果还要远——那剩下的候选者只会更远，不可能改善结果集，再搜就是浪费时间。

就好比你已经在手里攥着 3 个离你 10 米以内的人，现在名单上排第一的陌生人站 15 米远，那名单上剩下的只会更远，没必要找了。

上层 vs 第 0 层的区别

参数 上层（路由阶段） 第 0 层（精搜阶段）
ef 1 ef_（默认 16）
目的 找到 1 个路由节点即可 找到足够多的候选者
原因 上层稀疏，只需要”方向” 第 0 层稠密，需要精确结果

五、容易遇到的问题 / 常见疑问

Q1：为什么要层层往下推？为什么不在最高层直接找？

答：每一层的节点数量不同。在层 2 找到的最近节点 a，只是"层 2 里离 q 最近的"。到了层 1，节点变多了，a 旁边可能出现一个 b 比 a 更近。每往下一层，图更密，你能找到更好的入口。

所以层间下降相当于：**先用粗糙的图定位区域，再用精细的图锁定目标。**

Q2：ef_construction 和 ef 有什么区别？

参数 用在 作用
ef_construction 插入时 插入时搜多宽 → 影响建图质量
ef 查询时 查询时搜多宽 → 影响查询精度

两者越大：

  • ✅ recall 越高（找到真正近邻的概率越大）
  • ❌ 速度越慢

Q3：为什么上层用 ef=1 就够了？

因为上层节点极其稀疏。层 2 可能只有几十个节点，层 1 几百个。在这些层上你不需要找到很多候选——你只需要”走对方向”，到了第 0 层再展开。

Q4：如果最高层只有 entry_point 一个节点，search_layer 怎么走？

走不了——adjacency_[ep][level] 是空的。但没关系：

  • search_layer 会返回 [(d(ep), ep)]
  • results[0].second = ep
  • current_ep 不变
  • 继续往下一层走

所以唯一节点唯一节点即使没有邻居，它自己就是一个有效的路由节点。

Q5：如何提高 recall？

典型调参路径：先固定 M=16，调 ef_construction（100300）让 recall 达标，再用 ef（16200）做查询时的速度/精度权衡。

Q6：visited_tag_ 是什么？为什么用 tag 而不是清空数组？

不用 std::fill 清空的原因：visited_ 数组长度 = 向量总数（可能百万级）。每次搜索都清一遍是 O(N) 的，太慢。

Tag 方案的妙处：

  • visited_tag_ 从 0 开始，每次 +1
  • 数组初始化为 0
  • visited_[i] == visited_tag_ 等价于”本次是否访问过”
  • 每次搜索只 O(1) 增加一个计数器，不清数组

⚠️ 但有个坑：visited_tag_ 是 int，长期运行会溢出。实践中会在溢出时做一次全量重置，VecFlow 当前版本尚未处理这个边界情况。

六、代码结构速查

函数 所在行 作用 返回
random_level() ~121 为新节点分配层级 int（层级）
add() ~57 插入入口 void
add_impl() ~244 完整的多层插入逻辑 void
search() ~89 搜索入口 vector<SearchResult>
search_impl() ~307 完整的层间下降搜索 vector<SearchResult>
search_layer() ~132 单层贪心 BFS 搜索 vector<pair<float, size_t>>
select_neighbors() ~198 启发式邻居选择（多样性剪枝） vector<size_t>

回顾关键直觉：

插入时：从顶向下找入口 → 每层搜 ef_construction 个候选 → 启发式选邻居 → 双向连边

搜索时：从顶向下路由（ef=1）→ 在第 0 层精搜（ef=ef_）→ 返回 top-k

整个算法 = 多层图 + 贪心 BFS + 提前终止 + 多样性剪枝
