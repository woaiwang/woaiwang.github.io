CLAUDE.md 是一个特殊文件，Claude 在每次对话开始时都会读取它。你可以通过跑 /init 生成初稿，它会自动分析你的代码库并生成基础配置。

二、Vibe Coding 核心心法在 Claude Code 中的映射

Claude Code 不是一个“能生成代码的聊天界面”，而是一个能规划工作、读取代码库、跨目录编辑文件、运行测试并创建 PR 的 Agentic（智能体）工具。

2.1 核心心法映射表

描述意图，不描述步骤 自然语言 → 多步骤 Agentic 自动执行
先让它跑起来 Auto Mode + 宽松权限
保持心流 会话恢复 (-c) + CLAUDE.md 持久记忆
快速迭代 /rewind + /fork 回滚与分支实验
信任工具 --dangerously-skip-permissions（仅限受控 CI 环境）

2.3 让 Claude 先“采访”你，再动手

对于大型功能，别直接叫它动手写代码。从最简短的 Prompt 开始，让它反向收集需求：

我想构建 [简短描述]。
请使用 AskUserQuestion 工具详细采访我。
询问技术实现、UI/UX、边界情况、顾虑和权衡。不要问显而易见的问题，深入挖掘我可能没考虑到的难点。
持续采访直到我们覆盖所有内容，然后将完整规格写入 SPEC.md。
规格完成后，我们将开启一个新 session 来执行它。

2.4 Plan Mode — 大改动前的安全网

对生产关键文件或数据库迁移，先用 Plan Mode 让 Claude 处于只读状态。它会分析代码并给出修改计划，但绝不动手。

三、进阶与硬核操作（核心干货）

3.1 精准控制 Context，避免 Token 浪费

Context 管理是 Vibe Coding 的最核心技能。

  • /compact — Token 救星
    当上下文超过 80% 时，用它把历史记录压缩成密集摘要。
    用法：> /compact Focus on the auth module and current test failures
  • /fork — 对话层面的 Git Branch
    想尝试一种激进的重构方式？用 /fork try-aggressive-approach 创建对话分支。如果失败了，直接回到原来的 session，不破坏工作状态。
  • /rewind — 后悔药
    双击 Esc 或输入 /rewind。你可以选择“仅回滚代码”，在保留对话（诊断过程）的同时撤销所有文件修改。再也不用 git stash 了。

四、避坑指南（Anti-patterns）

❌ 坑 1：在一个超长 Session 里干所有事

新手喜欢开一个 session，加功能、改 bug、写测试全在里面。结果导致上下文被严重压缩，Claude 开始遗忘三小时前的架构决策。
✅ 大佬的做法：
每个独立任务使用 /clear 开新 session。依赖 CLAUDE.md 保留永久架构知识，使用 /compact 进行定向压缩。

❌ 坑 2：Prompt 太模糊，导致瞎猜

新手：“帮我优化这个函数”。Claude 不知道优化指的是性能、可读性还是内存，大概率改错。
✅ 大佬的做法（目标 + 约束 + 验证）：

优化 src/user.ts 中的 getUserData 函数。
目标：减少数据库查询次数（解决 N+1）。
约束：保持函数签名不变；不引入新依赖。
验证：必须通过 bun run test src/user.test.ts。完成后告诉我时间复杂度的变化。

❌ 坑 3：生产环境不控制权限

让不受约束的 AI 读取环境文件或修改 git 历史。
✅ 大佬的做法：
使用 --allowedTools 严格限制单次 session。
claude --allowedTools "Read,Glob,Grep,Bash(git log*)"（打造一个纯探索、零破坏风险的 session）。

五、完整 Vibe Coding 实战 Workflow

案例：用 Claude Code 从零排查并修复 Node.js 内存泄漏（OOM 崩溃）。

Phase 1：探索阶段（不污染主上下文）

Phase 2：管道收集运行时数据

Phase 3：Plan Mode 审查（零风险）

Phase 4：Worktree 并行修复

Phase 5：Review 与合并

🔁 Workflow 总结：

自然语言描述问题 → 子智能体探索 → Bash 管道查数据 → Plan Mode 零风险审查 → Worktree 并行实现 → Hooks 自动跑测试 → 子智能体 Code Review → 完美合入。

🚀 快速参考卡 (Cheat Sheet)

💡 终极心法：Claude Code 不是“更聪明的 Copilot”，它是一个通过权限、Hooks、子智能体和 Git Worktrees 分层系统来执行任务的自动化终端。Vibe Coding 的精髓就是：你负责定义意图和审查，Claude 负责脏活累活。
