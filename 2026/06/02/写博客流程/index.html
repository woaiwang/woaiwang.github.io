✨ 我的 Hexo 博客写作与发布 SOP ✨

这是一份从灵感到发布的完整操作流程，旨在帮助你高效、规范地管理你的个人博客。

阶段一：✍️ 灵感构思与准备

  • 🎯 核心目标：捕捉想法，明确主题，规划结构。

  1. 💡 捕捉灵感 (Capture)

    • 时机：学习新知、解决Bug、阅读好文、项目复盘后。
    • 工具：手机备忘录、Notion、ideas.txt 等。
    • 记录：仅记录核心标题或关键词，例如：
      • TCP三次握手比喻
      • Maven依赖冲突排查过程
      • 给新手的Git入门指南

  2. 🗺️ 构思大纲 (Outline)

    • 在开始写作前，花 3 分钟用 Markdown 标题 (#, ##, ###) 列出文章结构。
    • 示例：《深入理解Git工作流》

阶段二：💻 本地写作与预览

  • 🎯 核心目标：完成文章撰写，并在本地预览确认效果。

  1. 🚀 新建文章 (New Post)

    • 打开 PowerShell 或其他终端，cd 到你的博客根目录。

    • 执行 hexo new 命令，使用全英文或拼音作为文件名（利于SEO）。

    • 此命令会在 source/_posts/ 目录下创建对应的 .md 文件。

  2. ✍️ 撰写内容 (Writing)

    • 使用 VS Code 打开新创建的 .md 文件。

    • **认真填写 Front-matter**（文件头部的元数据），这是文章的“身份证”。

    • 使用标准的 Markdown 语法进行写作，专注于内容本身。

  3. 🔬 本地预览 (Preview)

    • 在终端中运行以下命令，启动本地服务器。
    • 命令运行后，终端会提示 Hexo is running at http://localhost:4000/。
    • 打开浏览器，访问 http://localhost:4000 即可实时预览。
    • 高效技巧：写作时，每次在 VS Code 中保存 (Ctrl + S)，浏览器中的页面会自动刷新（或手动刷新），方便你即时查看排版效果。

阶段三：🌐 发布上线

  • 🎯 核心目标：将本地的最新内容安全、完整地发布到互联网。

  1. 🛑 停止预览

    • 回到终端窗口，按下 Ctrl + C 停止本地服务器。

  2. 🚀 一键部署 (Deploy)

    • 执行以下黄金命令，完成清理、生成、部署三项全能操作。

    • 命令解析：

      • hexo clean: 清理。删除 public 文件夹，清除旧的缓存，避免潜在的冲突。
      • &&: 连接符。表示前一个命令成功执行后，再执行下一个。
      • hexo g: 生成 (generate)。根据源文件生成最新的静态网站文件到 public 文件夹。
      • hexo d: 部署 (deploy)。将 public 文件夹中的内容推送到 GitHub Pages。

阶段四：📢 发布后检查与推广

  • 🎯 核心目标：确认线上效果，并让你的知识分享给更多人。

  1. ✅ 线上验证 (Verify)

    • 部署命令成功后，等待 1-2 分钟（给 GitHub Pages 反应时间）。
    • 打开你的线上博客地址：https://你的用户名.github.io
    • 关键一步：使用强制刷新快捷键 Ctrl + F5 (Windows) 或 Cmd + Shift + R (Mac)，确保你看到的是最新的网站内容，而非浏览器缓存。
    • 检查新文章的标题、内容、格式、图片是否都显示正常。

  2. 🎉 分享推广 (Promote) - 可选

    • 复制你新文章的链接。
    • 分享到各大技术社区（如掘金、CSDN、SegmentFault）、社交媒体或技术交流群。
    • 引流技巧：在其他平台发布时，于文末附上你的博客原文链接，例如：

      “原文首发于我的个人博客，欢迎访问交流！”
